### 문제 설명

<p>자연수 N과 M이 주어졌을 때, 아래 조건을 만족하는 길이가 M인 수열을 모두 구하는 프로그램을 작성하시오.</p>

<ul>
	<li>1부터 N까지 자연수 중에서 중복 없이 M개를 고른 수열</li>
</ul>

### 입력 

 <p>첫째 줄에 자연수 N과 M이 주어진다. (1 ≤ M ≤ N ≤ 8)</p>

### 출력 

 <p>한 줄에 하나씩 문제의 조건을 만족하는 수열을 출력한다. 중복되는 수열을 여러 번 출력하면 안되며, 각 수열은 공백으로 구분해서 출력해야 한다.</p>

<p>수열은 사전 순으로 증가하는 순서로 출력해야 한다.</p>

 ### 🧠 배운 점 & 설계 과정
1. **문제 해석**: 
   - 자연수 N과 M이 주어졌을 때, 1부터 N까지 자연수 중에서 중복 없이 M개를 고른 수열을 모두 구하라.
   - 백트래킹의 개념이 처음 등장한다.
2. **설계**:
   - 함수의 인수를 depth로 받는다.
   - 골라낸걸 담을 picked 리스트 크기를 미리 정해놓는다.(M개를 뽑는것 이므로 크기도 M)
   - 결과를 담아낼 result 리스트와 사용했는지 여부를 판단할 nums 리스트를 생성한다.
   - depth가 M과 같아졌을때는 picked에 M개의 수가 저장된 상태이기 때문에 result 리스트에 추가하고 함수를 종료한다.
   - for문으로 1부터 N까지 순환하면서 이미 사용한 수면 넘기고 사용하지 않은 수면 picked에 추가한다.
   - depth가 0일때 첫번째 수를 picked에 담고 depth를 M까지 추가하면서 계속 담는다.
   - 끝까지 채우면 끝까지 채우기 전으로 돌아와 다른 수를 채운다.
   - 4,2를 예시로 들면 depth가 0일때 1부터 집어넣는다. 그리고 depth가 1이되면 이미 사용한 1빼고 그 다음 수인 2를 넣는다.
   - depth가 M과 같아졌으므로 1,2를 result에 넣고 depth가 1인 상태로 돌아간다.(백트래킹)
3. **실수 & 해결**:
   - 백트래킹의 개념에 대해 학습했다.
   - 할 수 있는 경우의수의 끝까지 가본 후에 다시 돌아와서 다른 경우의수를 찾는다는게 좀 어색했다.
