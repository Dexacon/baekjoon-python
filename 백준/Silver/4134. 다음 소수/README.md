### 문제 설명

<p>
	정수 n(0 ≤ n ≤ 4*10<sup>9</sup>)가 주어졌을 때, n보다 크거나 같은 소수 중 가장 작은 소수 찾는 프로그램을 작성하시오.</p>

### 입력 

 <p>
	첫째 줄에 테스트 케이스의 개수가 주어진다. 각 테스트 케이스는 한 줄로 이루어져 있고, 정수 n이 주어진다.</p>

### 출력 

 <p>
	각각의 테스트 케이스에 대해서 n보다 크거나 같은 소수 중 가장 작은 소수를 한 줄에 하나씩 출력한다.</p>

---
### 🧠 배운 점 & 설계 과정
1. **문제 해석**: 
   - 입력한 숫자보다 크거나 같은 수중 가장 작은 소수를 찾기
   - 정수론의 소수판정을 이용해 입력한 수가 소수인지 아닌지 판별
   - 브루트포스를 활용하여 소수가 나올 때까지 입력한 숫자보다 크거나 같은 수가 소수인지 확인
2. **설계**:
   - 2부터 n-1까지 검사하는 대신 n의 제곱근+1까지만 검사하여 시간복잡도를 단축
   - 소수판정을 함수화하여 가독성을 향상하고 조기 종료를 통해 불필요한 연산 방지
3. **실수 & 해결**:
   - 처음에 정수론의 소수판정 개념을 몰라서 브루트포스로 모두 연산하려고함
   - 구글링을 통해 정수론의 주요 개념인 최대공약수(GCD), 최소공배수(LCM), 소수 판정법, 그리고 에라토스테네스의 체에 대해 학습함
