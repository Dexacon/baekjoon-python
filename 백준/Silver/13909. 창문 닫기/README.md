### 문제 설명

<p>서강대학교 컴퓨터공학과 실습실 R912호에는 현재 N개의 창문이 있고 또 N명의 사람이 있다. 1번째 사람은 1의 배수 번째 창문을 열려 있으면 닫고 닫혀 있으면 연다.  2번째 사람은 2의 배수 번째 창문을 열려 있으면 닫고 닫혀 있으면 연다. 이러한 행동을 N번째 사람까지 진행한 후 열려 있는 창문의 개수를 구하라. 단, 처음에 모든 창문은 닫혀 있다.</p>

<p>예를 들어 현재 3개의 창문이 있고 3명의 사람이 있을 때,</p>

<ol>
	<li>1번째 사람은 1의 배수인 1,2,3번 창문을 연다. (1, 1, 1)</li>
	<li>2번째 사람은 2의 배수인 2번 창문을 닫는다. (1, 0, 1)</li>
	<li>3번째 사람은 3의 배수인 3번 창문을 닫는다. (1, 0, 0)</li>
</ol>

<p>결과적으로 마지막에 열려 있는 창문의 개수는 1개 이다.</p>

### 입력 

 <p>첫 번째 줄에는 창문의 개수와 사람의 수 N(<strong>1 ≤ N ≤ 2,100,000,000</strong>)이 주어진다.</p>

### 출력 

 <p>마지막에 열려 있는 창문의 개수를 출력한다.</p>

 ### 🧠 배운 점 & 설계 과정
1. **문제 해석**: 
   - 21억보다 작고 1보다 큰 숫자 N을 입력받는다.
   - N개의 창문을 1번째 사람부터 N번째 사람까지 자기 번호의 배수인 창문 열려있으면 닫고 닫혀있으면 연다.
   - N번째 사람까지 진행한 후 열려있는 창문의 개수를 구한다.
   - 단순히 N까지 반복해서 모든 경우를 진행하면 최대 반복 수는 21억 회이므로 시간이 초과된다.
2. **설계**:
   - N을 인수분해 했을 때 인수의 개수가 짝수이면 닫혀있고, 홀수이면 열려있다.
   - 따라서 인수의 개수가 홀수인 숫자의 개수만 구하면 열려있는 창문의 개수를 구할 수 있다.
   - 인수의 개수가 홀수인 수는 어떤 수의 제곱을 인수로 가지는 완전제곱수이므로 int(N**0.5)가 열려있는 창문의 개수이다.
   - 최대가 루트 21억 이기 때문에 시간 안에 구할 수 있다.
3. **실수 & 해결**:
   - 처음에는 완전 반복으로 풀었다가 시간이 초과됨
   - 그 후 인수의 개수가 홀수면 열려있다는 개념은 알아차렸으나 그 수가 완전제곱수를 의미하는 것을 알아차리지 못함
   - 완전제곱수임을 알아차린 후 루트를 이용하여 해결
