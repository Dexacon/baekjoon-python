### 문제 설명

<p>오늘도 서준이는 동적 프로그래밍 수업 조교를 하고 있다. 아빠가 수업한 내용을 학생들이 잘 이해했는지 문제를 통해서 확인해보자.</p>

<p>오늘은 <em>n</em>의 피보나치 수를 재귀호출과 동적 프로그래밍으로 구하는 알고리즘을 배웠다. 재귀호출에 비해 동적 프로그래밍이 얼마나 빠른지 확인해 보자. 아래 의사 코드를 이용하여 <em>n</em>의 피보나치 수를 구할 경우 코드1 코드2 실행 횟수를 출력하자.</p>

<p>피보나치 수 재귀호출 의사 코드는 다음과 같다.</p>

<pre>fib(n) {
    if (n = 1 or n = 2)
    then return 1;  # 코드1
    else return (fib(n - 1) + fib(n - 2));
}</pre>

<p>피보나치 수 동적 프로그래밍 의사 코드는 다음과 같다.</p>

<pre>fibonacci(n) {
    f[1] <- f[2] <- 1;
    for i <- 3 to n
        f[i] <- f[i - 1] + f[i - 2];  # 코드2
    return f[n];
}</pre>

### 입력 

 <p>첫째 줄에 <i>n</i>(5 ≤ <em>n</em> ≤ 40)이 주어진다.</p>

### 출력 

 <p>코드1 코드2 실행 횟수를 한 줄에 출력한다.</p>


  ### 🧠 배운 점 & 설계 과정
1. **문제 해석**: 
   - 피보나치수를 구하는 함수를 재귀의 방식과 동적 프로그래밍 방식으로 구현하고 둘의 실행횟수를 출력하라.
2. **설계**:
   - 문제에 명시된 의사 코드를 따라 재귀 방식의 함수와 동적 프로그래밍 방식의 함수를 만든다.
   - 동적 프로그래밍 방식은 함수 내부에서 for문으로 3부터 n까지 실행하여 피보나치 수를 구하므로 실행 횟수는 n-2이다.
   - 재귀함수는 return 1이 발생할 때마다 한번 실행된 것 이므로 피보나치 수와 실행 횟수가 동일하다.
   - 따라서 fibonacci(n),n-2를 출력한다.
3. **실수 & 해결**:
   - cnt로 시행횟수를 카운트 하려고 했으나 시간이 초과됨.
   - 동적 프로그래밍 방식 함수로 피보나치 수를 구해서 시간을 단축시켰다.
